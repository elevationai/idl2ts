# @elevationai/idl2ts

CORBA IDL to TypeScript compiler that generates type-safe TypeScript interfaces and stubs from CORBA IDL files.

## Quick Start

```bash
# No installation needed - use npx directly
npx @elevationai/idl2ts compile myservice.idl -o ./generated
```

## Installation

For regular use, you can install globally:

```bash
npm install -g @elevationai/idl2ts
```

Or as a development dependency:

```bash
npm install --save-dev @elevationai/idl2ts
```

## Usage

### Command Line

```bash
# Using npx (recommended - no global install needed)
npx @elevationai/idl2ts compile <input.idl> -o <output-directory> [options]

# Or if installed globally
idl2ts compile <input.idl> -o <output-directory> [options]
```

#### Options

- `-o, --output <path>` - Output directory for generated TypeScript files
- `-I, --include <paths...>` - Include paths for IDL files
- `--no-stubs` - Do not generate client stubs
- `--skeletons` - Generate server skeletons
- `--no-helpers` - Do not emit helper functions
- `--corba-import <path>` - Import path for CORBA library (default: "corba")
- `-v, --verbose` - Verbose output

#### Examples

```bash
# Compile a single IDL file (using npx)
npx @elevationai/idl2ts compile myservice.idl -o ./generated --corba-import corba

# Compile with include paths
npx @elevationai/idl2ts compile myservice.idl -o ./generated -I ./idl-includes --corba-import @myorg/corba

# Compile multiple files using glob patterns
npx @elevationai/idl2ts compile "idls/*.idl" -o ./generated

# If installed globally
idl2ts compile myservice.idl -o ./generated
```

### Programmatic API

```typescript
import { IDLCompiler } from '@elevationai/idl2ts';

const compiler = new IDLCompiler({
  outputPath: './generated',
  includeStubs: true,
  includeSkeletons: false,
  emitHelpers: true,
  corbaImportPath: 'corba',
  verbose: false
});

compiler.compile('myservice.idl');
```

## Features

- **Full CORBA IDL Support**: Handles modules, interfaces, structs, unions, enums, typedefs, exceptions, and more
- **Cross-Module Type Resolution**: Correctly resolves types across module boundaries with proper qualified names  
- **Nested Type Support**: Handles nested types (enums, structs, etc.) defined within interfaces
- **Type-Safe Async/Await**: Generates modern TypeScript with async/await patterns
- **CORBA-Compliant Stub Generation**: Generates client stubs following CORBA specification with proper request handling
- **Exception Handling**: IDL exceptions become TypeScript classes extending `CORBA.SystemException`
- **Inheritance Support**: Properly handles interface inheritance hierarchies

## IDL to TypeScript Mapping

### Basic Types

| IDL Type | TypeScript Type |
|----------|----------------|
| `short` | `number` |
| `long` | `number` |
| `long long` | `bigint` |
| `unsigned short` | `number` |
| `unsigned long` | `number` |
| `unsigned long long` | `bigint` |
| `float` | `number` |
| `double` | `number` |
| `char` | `string` |
| `wchar` | `string` |
| `string` | `string` |
| `wstring` | `string` |
| `boolean` | `boolean` |
| `octet` | `number` |
| `any` | `any` |
| `void` | `void` |

### Complex Types

- **sequence<T>** → `T[]`
- **T[N]** → `T[]` (fixed-size arrays)
- **struct** → TypeScript interface
- **enum** → TypeScript enum
- **union** → TypeScript union type with discriminator
- **exception** → TypeScript class extending `CORBA.SystemException`
- **interface** → TypeScript interface extending `CORBA.ObjectRef` with async methods

### Example IDL

```idl
module Example {
  interface Calculator {
    long add(in long a, in long b);
    double divide(in double numerator, in double denominator) 
      raises (DivisionByZero);
  };
  
  exception DivisionByZero {
    string message;
  };
};
```

### Generated TypeScript

```typescript
/**
 * This file was automatically generated by idl2ts
 * DO NOT EDIT THIS FILE DIRECTLY
 */

import { CORBA } from "corba";

export interface Calculator extends CORBA.ObjectRef {
  add(a: number, b: number): Promise<number>;
  divide(numerator: number, denominator: number): Promise<number>;
}

export class Calculator_Stub implements Calculator {
  constructor(private objRef: CORBA.ObjectRef) {}
  
  async add(a: number, b: number): Promise<number> {
    const request = this.objRef.create_request("add");
    request.add_in_arg("a", a);
    request.add_in_arg("b", b);
    await request.invoke();
    return request.return_value();
  }
  
  async divide(numerator: number, denominator: number): Promise<number> {
    const request = this.objRef.create_request("divide");
    request.add_in_arg("numerator", numerator);
    request.add_in_arg("denominator", denominator);
    await request.invoke();
    return request.return_value();
  }
}

export class DivisionByZero extends CORBA.SystemException {
  message: string;

  constructor(message: string) {
    super('DivisionByZero');
    this.message = message;
  }
}
```

### Testing

The project includes a comprehensive test suite:

- **Unit Tests**: Test individual components (parser, generator, preprocessor)
- **Integration Tests**: Test end-to-end compilation scenarios
- **Edge Cases**: Test error handling and unusual IDL constructs
- **Fixtures**: Real-world IDL examples for testing

Test files are located in the `test/` directory:
- `test/unit/` - Unit tests for individual components
- `test/integration/` - Integration tests for the full compiler
- `test/fixtures/` - IDL fixture files used in tests
- `test/helpers/` - Test utilities and helpers
